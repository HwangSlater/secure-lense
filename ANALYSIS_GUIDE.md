## 🔍 일반 분석 동작 방식

일반 분석은 파일을 업로드하면 자동으로 실행되며, 여러 보안 엔진과 분석 툴을 순차적으로 실행하여 종합적인 위협 평가를 수행합니다.

### 1. 파일 검증 단계

**목적**: 업로드된 파일의 기본 검증

- **파일 크기 확인**: 최대 50MB 제한
- **확장자 검증**: 허용된 확장자만 허용 (.exe, .dll, .pdf, .docx, .eml, .zip)
- **파일명 검증**: 경로 탐색 공격 방지 (.., /, \ 제거)

### 2. ClamAV 바이러스 스캔

**목적**: 알려진 바이러스 시그니처 탐지

**동작 방식**:
1. ClamAV 데몬에 소켓 연결 (포트 3310)
2. `SCAN` 명령 전송
3. 응답 파싱: "FOUND" 키워드 확인
4. 탐지 시 바이러스 이름 반환

**탐지 내용**:
- 알려진 악성코드 시그니처
- 트로이 목마, 웜, 랜섬웨어 등

**위험도 영향**: +40점

### 3. YARA 규칙 매칭

**목적**: 패턴 기반 악성코드 특징 탐지

**동작 방식**:
1. `backend/rules/` 디렉토리에서 모든 `.yar` 파일 로드
2. YARA 엔진으로 파일 스캔
3. 매칭된 규칙 이름 반환

**탐지 내용**:
- 쉘코드 패턴
- 자격 증명 탈취 행위
- 한국어 피싱 키워드
- 이중 확장자 파일

**위험도 영향**: +30점 (최대)

### 4. 엔트로피 분석

**목적**: 파일 패킹/암호화 탐지

**동작 방식**:
1. 파일의 바이트 빈도 계산
2. Shannon 엔트로피 공식 적용: `H = -Σ(p(x) * log2(p(x)))`
3. 0-8.0 척도로 측정

**해석**:
- **0-6.5**: 정상적인 파일 (텍스트, 일반 바이너리)
- **6.5-7.0**: 약간의 압축 또는 패킹 가능성
- **7.0-7.5**: 높은 엔트로피, 패킹 또는 압축 가능성 높음
- **7.5-8.0**: 매우 높은 엔트로피, 암호화 또는 강력한 패킹 가능성

**위험도 영향**:
- 7.5 이상: +20점
- 7.0-7.5: +15점
- 6.5-7.0: +10점

### 5. 파일 타입 검증

**목적**: 확장자 위조 탐지

**동작 방식**:
1. `python-magic` 라이브러리로 실제 파일 시그니처 확인
2. 확장자와 실제 파일 타입 비교
3. 불일치 시 경고

**탐지 내용**:
- `.pdf.exe` 같은 이중 확장자
- 실제 실행 파일인데 `.pdf` 확장자
- 실제 ZIP 파일인데 `.docx` 확장자

**위험도 영향**: +15점 (불일치 시)

### 6. 파일 해시 계산

**목적**: 파일 고유 식별자 생성 및 위협 인텔리전스 연동

**동작 방식**:
1. 파일을 4KB 청크 단위로 읽기
2. MD5, SHA1, SHA256 해시 동시 계산
3. 해시값 반환

**용도**:
- **파일 고유 식별**: 동일 파일 여부 확인
- **VirusTotal 연동**: SHA256 해시로 알려진 악성코드 검색
- **MalwareBazaar 연동**: SHA256 해시로 악성코드 샘플 데이터베이스 검색
- **파일 무결성 검증**: 파일 변조 여부 확인

**해시 알고리즘**:
- **MD5**: 128비트 해시 (빠른 검색용)
- **SHA1**: 160비트 해시 (호환성용)
- **SHA256**: 256비트 해시 (보안 강화, 표준)

**위험도 영향**: 직접적인 점수 부여 없음 (외부 API 연동에 사용)

### 7. 파일 크기 이상 탐지

**목적**: 비정상적인 파일 크기로 드로퍼, 데이터 유출, 과도한 페이로드 탐지

**동작 방식**:
1. 파일 크기 측정 (바이트 단위)
2. 파일 타입별 정상 크기 범위와 비교
3. 이상 징후 탐지

**탐지 내용**:

**매우 작은 파일 (1KB 미만)**:
- 불완전한 파일 또는 드로퍼 가능성
- 악성코드가 추가 데이터를 다운로드하는 경우

**매우 큰 파일 (100MB 초과)**:
- 데이터 유출 시도 가능성
- 과도한 페이로드 포함 가능성
- 정상적인 문서 파일보다 비정상적으로 큰 경우

**실행 파일 크기 이상**:
- PE 파일이 10KB 미만: 비정상적으로 작은 실행 파일
- 일반적으로 실행 파일은 최소 20KB 이상

**위험도 영향**: +10점 (최대, 이상 징후 개수에 따라)

### 8. 파일명 패턴 분석

**목적**: 의심스러운 파일명 패턴으로 피싱 및 악성코드 탐지

**동작 방식**:
1. 파일명 문자열 분석
2. 정규식 패턴 매칭
3. 의심 패턴 탐지

**탐지 내용**:

**이중 확장자**:
- `invoice.pdf.exe`, `document.docx.bat` 등
- 사용자가 확장자를 잘못 인식하도록 유도
- Windows는 마지막 확장자만 인식하므로 실제로는 실행 파일

**의심스러운 키워드**:
- 영어: `invoice`, `receipt`, `document`, `scan`, `photo`, `urgent`, `important`, `readme`, `install`, `update`
- 한국어: `긴급`, `중요`, `확인`, `인증`, `업데이트`, `설치`
- 피싱 공격에서 자주 사용되는 단어들

**랜덤 파일명**:
- 20자 이상의 랜덤 문자로 구성된 파일명
- 악성코드가 자동 생성한 파일명 패턴

**과도한 공백**:
- 파일명에 공백이 3개 이상
- 피싱 이메일에서 자주 사용되는 패턴

**유니코드 문자**:
- ASCII 범위를 벗어난 문자 포함
- 호모그래프 공격 (예: `а` (키릴 문자) vs `a` (라틴 문자))
- 사용자가 실제 URL과 혼동하도록 유도

**위험도 영향**: +15점 (최대, 이상 징후 개수에 따라)

### 9. Base64 인코딩 탐지

**목적**: 파일 내 Base64 인코딩된 데이터로 난독화 및 페이로드 숨김 탐지

**동작 방식**:
1. 파일을 바이너리로 읽기
2. Base64 패턴 정규식 매칭 (`[A-Za-z0-9+/]{20,}={0,2}`)
3. 발견된 문자열을 Base64 디코딩 시도
4. 유효한 Base64 문자열만 수집
5. 디코딩된 데이터 크기 확인

**탐지 내용**:

**Base64 인코딩 문자열**:
- 20자 이상의 Base64 패턴
- 유효한 Base64 디코딩 가능한 문자열
- 디코딩된 데이터가 10바이트 이상

**의심스러운 패턴**:
- **다수의 Base64 문자열 (5개 이상)**: 난독화 또는 페이로드 숨김 가능성
- **대용량 Base64 데이터 (1KB 이상)**: 큰 페이로드를 숨기려는 시도
- **연속된 Base64 블록**: 전체 파일이 Base64로 인코딩된 경우

**용도**:
- 악성코드가 페이로드를 Base64로 인코딩하여 숨기는 경우 탐지
- Office 매크로나 스크립트에서 Base64로 인코딩된 명령어 탐지
- 이메일 본문에서 Base64로 인코딩된 악성 링크 탐지

**위험도 영향**: +15점 (최대, 이상 징후 개수에 따라)

### 10. 바이너리 분석

**목적**: 실행 파일의 의심스러운 패턴 탐지

**동작 방식**:
1. 파일을 바이너리로 읽기
2. 정규식 패턴 매칭으로 쉘코드 탐지
3. 의심스러운 문자열 추출

**탐지 내용**:

**쉘코드 패턴**:
- NOP sled (0x90 반복)
- JMP short (0xEB 0xFE)
- CALL/PUSH/XOR 명령어 시퀀스
- PUSH/POP 레지스터 초기화 패턴

**의심스러운 문자열**:
- `cmd.exe`, `powershell`, `wscript.exe`
- 레지스트리 조작: `HKEY_CURRENT_USER`, `reg add`
- 네트워크 API: `URLDownloadToFile`, `WinExec`
- 메모리 조작: `VirtualAlloc`, `CreateRemoteThread`
- Tor 네트워크: `.onion`, `tor2web`
- 의심스러운 URL 패턴

**위험도 영향**:
- 쉘코드 패턴: +20점
- 의심 문자열 5개 이상: +15점
- 의심 문자열 3개 이상: +10점

### 11. PE 파일 분석 (pefile)

**목적**: Windows 실행 파일 구조 분석

**동작 방식**:
1. PE 파일 헤더 파싱
2. 섹션 특성 분석
3. 임포트된 API 확인

**탐지 내용**:

**의심스러운 섹션**:
- 실행 가능 + 쓰기 가능한 섹션 (W+X)
- 패킹 섹션 이름: `.packed`, `.upx`, `.aspack`

**의심스러운 API 임포트**:
- `VirtualAlloc`, `VirtualProtect`: 메모리 조작
- `CreateRemoteThread`, `WriteProcessMemory`: 프로세스 주입
- `URLDownloadToFile`, `WinExec`: 다운로드 및 실행
- `SetWindowsHookEx`: 훅 설치
- `CreateService`: 서비스 등록

**섹션 수 이상**:
- 3개 미만: 패킹 가능성

**위험도 영향**: +15점

### 12. PE 강화 분석 (LIEF)

**목적**: LIEF 라이브러리를 사용한 상세 PE 분석

**동작 방식**:
1. LIEF로 PE 파일 파싱
2. 각 섹션의 엔트로피 계산
3. 임포트/익스포트 상세 분석
4. 섹션 특성 확인

**탐지 내용**:

**고엔트로피 섹션**:
- 섹션별 엔트로피 7.0 이상: 패킹 가능성

**의심스러운 섹션 특성**:
- 실행 가능 + 쓰기 가능: 코드 주입 가능성

**의심스러운 API**:
- 메모리 조작, 프로세스 주입, 네트워크 통신 관련 API

**위험도 영향**: +20점 (최대)

### 13. Office 문서 분석 (oletools)

**목적**: VBA 매크로 및 의심스러운 내용 탐지

**동작 방식**:
1. `olevba`로 VBA 매크로 추출
2. 매크로 코드 분석
3. 자동 실행 매크로 탐지
4. 의심스러운 키워드 검색

**탐지 내용**:

**매크로 정보**:
- 매크로 개수
- 자동 실행 매크로: `Auto_Open`, `Workbook_Open`, `Document_Open`

**의심스러운 VBA 키워드**:
- `Shell`, `CreateObject`: 외부 프로그램 실행
- `WScript.Shell`, `Exec`, `Run`: 명령 실행
- `DownloadFile`, `URLMon`, `XMLHTTP`: 파일 다운로드
- `RegWrite`, `RegDelete`: 레지스트리 조작
- `SendKeys`: 키보드 입력 시뮬레이션

**위험도 영향**:
- 자동 실행 매크로: +20점
- 일반 매크로: +15점
- 의심 키워드: +15점 (최대)

### 14. PDF 분석 (PyPDF2)

**목적**: PDF 파일의 의심스러운 요소 탐지

**동작 방식**:
1. PDF 구조 파싱
2. JavaScript 포함 여부 확인
3. 인터랙티브 요소 확인
4. 임베디드 파일 확인

**탐지 내용**:

**JavaScript**:
- PDF 내 JavaScript 코드: 악성 PDF의 일반적인 특징

**인터랙티브 요소**:
- 액션, 폼, 주석: 사용자 상호작용 유도

**임베디드 파일**:
- PDF 내 다른 파일 포함

**위험도 영향**:
- JavaScript: +15점
- 의심 객체: +10점 (최대)

### 15. ZIP 파일 분석

**목적**: 압축 파일 내부의 위험 요소 탐지

**동작 방식**:
1. ZIP 파일 구조 파싱
2. 내부 파일 목록 추출
3. 각 파일의 확장자 및 이름 분석

**탐지 내용**:

**이중 확장자 파일**:
- `invoice.pdf.exe`, `document.docx.bat` 등

**실행 파일**:
- `.exe`, `.bat`, `.cmd`, `.scr`, `.vbs`, `.js`, `.ps1`

**중첩 아카이브**:
- ZIP 내부의 ZIP, RAR, 7Z 등

**암호화**:
- 암호화된 파일 포함

**위험도 영향**:
- 의심 파일: +15점 (최대)
- 암호화: +10점
- 중첩 아카이브: +10점
- 이중 확장자: +15점 (최대)

### 16. 문자열 추출 강화

**목적**: 파일에서 URL, IP, 이메일 주소 추출

**동작 방식**:
1. ASCII 문자열 추출 (4자 이상)
2. Unicode (UTF-16 LE) 문자열 추출
3. 정규식으로 URL, IP, 이메일 패턴 매칭

**추출 내용**:
- URL: `http://`, `https://`로 시작하는 문자열
- IP 주소: IPv4 주소 패턴
- 이메일 주소: 이메일 형식 패턴

**용도**: C2 서버, 피싱 사이트, 연락처 정보 식별

### 17. 이메일 분석

**목적**: 스피어피싱 공격 탐지

**동작 방식**:
1. 이메일 헤더 파싱
2. From, Reply-To, Return-Path 비교
3. 본문에서 피싱 키워드 검색
4. URL 추출 및 분석
5. 첨부파일 이중 확장자 확인

**탐지 내용**:

**발신자 위조**:
- From과 Reply-To가 다른 경우

**피싱 키워드**:
- "긴급", "계좌 확인", "비밀번호 변경", "당첨", "세금 환급", "배송 확인", "본인 인증"

**의심스러운 URL**:
- 혼합 스크립트 (homograph 공격 가능성)

**이중 확장자 첨부파일**:
- `invoice.pdf.exe` 등

**위험도 영향**:
- 발신자 위조: +10점
- 피싱 키워드: +20점 (최대)
- 이중 확장자: +10점
- 의심 URL 3개 이상: +10점

### 18. 외부 API 분석

**목적**: 외부 위협 인텔리전스와 연동

**동작 방식**:

**파일 해시 계산**:
1. MD5, SHA1, SHA256 해시 계산

**VirusTotal 조회**:
1. SHA256 해시로 VirusTotal API 호출
2. 다수 백신 엔진 탐지 결과 확인
3. 탐지 비율 계산

**MalwareBazaar 조회**:
1. SHA256 해시로 MalwareBazaar API 호출
2. 악성코드 샘플 데이터베이스 검색

**URL 스캔** (파일 내 URL이 있는 경우):
1. 파일에서 URL 추출
2. URLScan.io로 각 URL 스캔 (최대 3개)

**IP 정보 조회** (파일 내 IP가 있는 경우):
1. 파일에서 IP 주소 추출
2. IP-API로 지오로케이션 정보 조회 (최대 5개)

**위험도 영향**:
- VirusTotal 50% 이상 탐지: +25점
- VirusTotal 20-50% 탐지: +15점
- VirusTotal 탐지: +10점
- MalwareBazaar 매칭: +20점
- 악성 URL: +15점 (최대)

### 19. 위험도 점수 계산

**목적**: 모든 분석 결과를 종합하여 0-100점 위험도 점수 계산

**계산 방식**:
1. 각 분석 항목의 점수를 합산
2. 최대 100점으로 제한
3. 점수에 따라 등급 결정:
   - 0-20: 매우 낮음
   - 21-40: 낮음
   - 41-60: 보통
   - 61-80: 높음
   - 81-100: 매우 높음

**저장**:
- 모든 분석 결과를 데이터베이스에 저장
- 분석 이력에서 확인 가능

## 📊 분석 결과 해석

### 위험도 등급별 의미

**매우 낮음 (0-20점)**:
- 안전한 파일로 판단
- 추가 조치 불필요
- 모니터링만 수행

**낮음 (21-40점)**:
- 일부 의심스러운 요소 발견
- 주의 깊게 확인 필요
- 신뢰할 수 있는 출처인지 확인

**보통 (41-60점)**:
- 악성 징후 다수 발견
- 실행 금지
- 격리 또는 삭제 권장

**높음 (61-80점)**:
- 악성 파일일 가능성 매우 높음
- 즉시 격리 또는 삭제
- 시스템 스캔 권장

**매우 높음 (81-100점)**:
- 악성코드로 판단
- 즉시 삭제 및 조치
- 감염 여부 확인 필요

### 주요 지표 해석

**ClamAV 탐지**:
- 알려진 악성코드 시그니처 매칭
- 신뢰도: 매우 높음
- 즉시 삭제 권장

**YARA 매칭**:
- 악성코드 패턴 매칭
- 신뢰도: 높음
- 추가 분석 권장

**높은 엔트로피 (7.0 이상)**:
- 파일이 패킹되었거나 암호화되었을 가능성
- 악성코드가 숨겨져 있을 수 있음
- 동적 분석 필요

**파일 타입 불일치**:
- 확장자 위조 시도
- 악성 파일을 안전한 파일로 위장
- 매우 의심스러움

**Office 자동 실행 매크로**:
- 문서 열기 시 자동 실행
- 매우 위험
- 매크로 비활성화 권장

**PDF JavaScript**:
- 악성 PDF의 일반적인 특징
- 자동 실행 스크립트 가능성
- PDF 뷰어 보안 설정 확인

**ZIP 이중 확장자**:
- 사용자를 속이기 위한 시도
- 실행 파일을 문서로 위장
- 매우 위험

**VirusTotal 다수 탐지**:
- 여러 백신 엔진에서 탐지
- 신뢰도 매우 높음
- 즉시 삭제 권장
